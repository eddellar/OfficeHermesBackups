备份失败，原因清晰：远程仓库的 commit history 中包含 `state-snapshots/*/state.db` 文件（176MB 和 194MB），远超 GitHub 100MB 限制，即使我们删除了这些文件，push 时那些历史 commit blob 仍然会被引用，导致 GitHub pre-receive hook 拒绝。

由于 `git-filter-repo` 未安装，无法 rewrite 历史来永久移除这些大文件。

**需要用户干预：**
1. 安装 `git-filter-repo`：`pip install git-filter-repo`
2. 或者在 GitHub 仓库设置中开启 Git LFS
3. 或者手动删除远程仓库重建
